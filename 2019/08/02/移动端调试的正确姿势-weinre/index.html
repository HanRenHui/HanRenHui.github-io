前言

最近在做一个移动端的项目, 第一次写移动端, 没啥经验, 调试全部基于谷歌浏览器的手机端模拟, 在项目接近尾声的时候就把项目部署到了阿里云上, 真机上看了看效果, 这不看不知道, 一看吓一跳, 那小心拔凉拔凉的: 样式很多是乱的, 有的数据没有加载出来, 用的别人的cdn图片全部加载不出😭, 页面滑动卡顿不流畅等等… 你们能想象我当时的心情么, 难受归难受, 还是得硬着头皮调试, 我当时在想手机端访问项目效果, 难道要我每次代码改完部署到云上, 然后真机查看效果么, 但我这bug不是一点半点啊, 这样搞, 不累死也烦死了, 于是就想能不能在本地开一个服务器, 然后手机访问本地的服务器, 在手机上调试页面效果, 然后赶紧开始按照自己的想法谷歌百度, 没想到真有👇得救了~~

weinre

手机端调试页面有很多姿势, 今天主要说下weinre, 也是这两天我在用的, 说说它的基本使用, 和我在使用的过程中遇到的坑

weinre简介

学习一个技术的使用之前, 总是要先看看这玩意是干啥的, 知道他的工作原理, 才能在使用的过程中得心应手👇

weinre: 全称是网页远程审查（Web Inspector Remote), 其他废话不多说了, 看了让人心烦, 一句话就是: 让我们在pc上调试手机端的页面(够不够简洁)

weinre 工作原理

借用一下网上用烂的两张原理图, 罪过罪过


是不是一脸懵? 懵就对了, 咱稍微解释下👇:
Debug客户端（client）：运行在pc上, 类似谷歌的F12开发者工具, 鼠标点到dom元素, 页面对应dom变蓝，远程调试客户端。
Debug服务端（agent）：本地的HTTPServer，为Debug目标页面与Debug客户端建立连接。
Debug目标页面（target）：简单来说就是你正在开发的页面。
客户端、目标页面与Debug服务端之间使用XMLHttpRequest (XHR)进行HTTP通信，通常的使用情形是将Debug客户端与服务端搭建在桌面开发环境，Debug目标页面放在移动设备。此时, 你的开发姿势就是, pc端检查元素, 查看对应dom, 此时变蓝的页面已经转移到手机上而不是pc, (后悔没买大屏手机, 9102年的iphone小6瑟瑟发抖, 等我有钱了我也要买有刘海的手机😕)
注意: 由于Weinre的debug客户端是基于Web Inspector开发，而Web Inspector只兼容WebKit核心的浏览器，所以只能在Chrome/Safari浏览器打开Weinre客户端进行调试

weinre的安装

很简单, 就是安装一个名为weinre的全局npm包, 命令如下👇

npm install weinre -g

让pc和手机位于同一网段

weinre安装完在使用之前, 我们需要让pc和手机位于同一网段, 方法很多, 比如说: 电脑连着宽度给手机开热点, 或者手机和电脑连接同一个路由器, 由于我手机电脑都是连接的wifi, 所以我使用的是第二种

weinre的使用

打开命令行: 输入指令weinre --boundHost [IP_address] --httpPort [port], 这里的IP_address 是你的pc IP地址 port是随意指定的端口, 那么我们如何查看pc IP地址呢? 很简单 打开命令行输入 ipconfig 会看到下面的界面👇：

那么哪个是电脑的ip地址呢, 网上一致的答案是(看了不下五篇不同的文章): 👇

但结果呢? 老子花了一晚上和一个上午硬是没搞定, 前前后后看了不下十篇文章, 回答一直都是这个地址, 直接导致我排查原因的时候没注意到这个点, 在我实在受不了的时候脑子灵光一现, 如果pc和手机是同一网段的, 那么他们的前三个数字通常情况下应该是一样的, 我查看了下手机的ip, 然后看了看这个地址, 这俩压根不是一个网段！！！ 那倒是哪个是正确的IP呢? 我眼神往上一飘, 果然, 看到了一个和手机ip前三位相同的ip, 就是它, 小哪吒👇


最后果然成功了, 撒花, 我选用的ip虽然起作用了, 但是网上基本上能查到的答案都说是之前那个ip, 难道是我的电脑特殊? 我感觉其中一定有哪个点我搞错了, 反正大家这俩ip都试试, 一个不行换另一个, 选择ip以后 关闭电脑的防火墙, 或者开启对应的端口, 这个自行百度, 还有一步: 我是基于webpack开发的, 还需要把webpack-dev-server 的HOST设为 ‘0.0.0.0’, 最后一步: 在你要调试的html页面里嵌入一个script标签, 就是它👇


全部操作之后, 输入对应ip, 我们就可以回车刚才的指令了, 如果看到以下显示, 则表示成功了

现在我们在手机端打开safari或者chrome, 地址栏输入 http://192.168.0.106:8080(ip和端口号换成你自己的, 这里的端口号指的是你项目本地服务器的端口, 不是给weinre设的端口) 就可以在手机端看到我们的页面了目前只是在手机端看到我们的页面, 那么我们怎么在pc上调试呢？ pc上输入 http://192.168.0.106:8081/, 同样ip和端口号换成你的(这里的端口就是你给weinre设的端口也就是刚才之前在命令行制定的httpPort), 之后应该会显示这样的一个页面
