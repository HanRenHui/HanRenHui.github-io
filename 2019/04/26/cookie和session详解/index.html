cookie的作用

As we all know, HTTP协议是无状态的, 所以服务端对于客户端的请求不会有任何的记忆, 到底是同一个客户端请求了多次, 还是请求来自不同的客户端, 这些都不得而知, 但有时候我们希望服务端能够’眼熟’客户端, 这样客户端在请求服务端的时候就能少做很多事情, 从来提高用户的体验, 常见的应用场景就是自动登陆, 回想一下, 我们在一个网站登陆过一次以后是不是有相当长的时间之内就不用再次登陆了, 这里应用的就是cookie

path

该字段用于设置cookie在哪个路径下有效

Expires 和 Max-Age

这两个字段的作用是一样的, 都是设置cookie的过期时间的, 不同的是 Expires使用的是绝对时间, Max-Age使用的相对时间,
简单来说就是Expires设置的是过期的具体时间, 而Max-Age 设置的是多长时间以后过期,如果不设置这俩字段, 则过期时间默认为当前会话, 也就是说你浏览器一关就没了 让我们看下这俩怎么设置

Size

标识cookie大小, 浏览器自动设置, 不归咱管
