react-redux中的组件

react-redux 将所有组件分为两大类，UI组件和容器组件，UI组件有如下特征：

  • 只负责UI展示，不带有任何业务逻辑
  • 无状态(state)
  • 所有数据甚至业务逻辑都由父级组件传入
  • 不使用任何redux的API

而容器组件则恰恰相反 负责管理state，处理各种业务逻辑，但是不负责UI展示, 如果一个组件既有UI又有业务的话怎么办呢？我们可以给这个组件外层包一个组件，外层组件我们将它定义成容器组件，内部组件的所有状态逻辑等都定义到 外层容器组件上，再由容器组件将数据和逻辑传入内层UI组件; 在react-redex中，UI组件由我们提供，然后将数据和逻辑封装在对象中，传入react-redex提供的connect方法中，该方法会自动给我们在ui组件外层套一个容器组件，传入的数据(mapStateToProps)和逻辑对象(mapDispatchToProps)则会被传入到内层UI组件中

connect实现

但是上边的connect实现还不太优雅，因为里面依赖死了store，我们需要做一下优化, 我们可以将store定义在src文件下的index.js中，再通过context让connet模块直接拿到store, 这时我们就需要在顶层定义一个provider组件，该组件没有其他作用，就是通过context给connect分发store的
